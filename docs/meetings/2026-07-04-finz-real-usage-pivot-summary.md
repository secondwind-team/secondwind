## 핵심 결론

현재까지의 개발이 기능 테스트 중심이었다는 공감대 하에, **실사용 중심으로 전환**하기로 방향을 잡았다. 핵심 축은 **우정주 추천 → 선택 → 단기 목표 설정 → 데일리 리포트** 흐름으로 구체화되었으며, 관련 기반 기능(포트폴리오)은 이미 일부 구현된 상태다.

## 결정 사항

- **우정주 추천 방식**: 핀즈가 기간·목표 수익률·카테고리 조건을 입력받아 후보 종목 리스트를 제공 → 팀이 대화로 합의하여 확정
- **단기 목표 설정**: 우정주 확정 시 매수가, 손절가, 목표 수익률(예: 10~20%), 투자 기간을 함께 설정
- **데일리 리포트 자동화**: 우정주 등록 후 가격·뉴스·차트 정보를 정기적으로 채팅방에 자동 발송
- **스레드 모드 배포 완료**: 슬랙 방식의 스레드 UI가 이미 배포되어 채팅방 모드 전환 가능
- **토큰 최적화는 후순위**: 개발 단계에서는 토큰 제한 없이 기능 우선 구현, 최적화는 나중에

## 논의된 주요 아이디어

- **스레드 구조 도입**: 슬랙처럼 메시지 자체를 주제 단위로 구성해 대화 휘발성 감소 및 AI의 주제별 분류 용이
- **모의투자 대회**: 기간·수익률 기준으로 등수를 매기는 대회 형식 → 피드 기능과 연동해 결과 공유
- **AI 자동 카테고리 분류**: 대화 내 종목 언급을 AI가 자동으로 분류·요약하여 관심도 높은 종목 도출
- **컨텍스트 엔지니어링 개선 필요**: 현재 LLM 호출 시 이전 맥락 전달 구조가 미흡하며, 스냅샷 방식의 요약 누적 저장 검토
