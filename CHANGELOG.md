# Changelog

secondwind 의 주요 변경 사항을 기록합니다. 날짜 포맷은 `YYYY-MM-DD`, 버전은 4자리 `MAJOR.MINOR.PATCH.MICRO`.

## [0.1.33.1] - 2026-07-05

### Changed
- **첨부 이미지·파일을 비공개(private)로 전환 + 방 멤버 게이트 프록시.** 0.1.33.0 의 첨부는 world-readable URL(누구나 URL 만 알면 열람)이었는데, 채팅 사진 프라이버시를 위해 **기본 비공개**로 바꾸고 열람을 방 멤버만 통과하는 프록시(`/api/finz/party/[id]/attachment`)로 게이트했다. 메시지엔 URL 대신 blob pathname 만 저장하고, 프록시가 *로그인 · 방 멤버 · 그 방에 올라온 첨부인지*를 확인한 뒤에만 스트리밍한다. SSRF 방어(pathname 은 bare path 만) + 오리진 XSS 방어(비이미지는 강제 다운로드 + `nosniff`). **Blob 스토어는 Private 로 생성**한다. 아직 스토어 미연결이라 사용자 영향 없음(코드 정비).

## [0.1.33.0] - 2026-07-05

### Added
- **메시지 URL 자동 링크 + 링크 미리보기 썸네일.** 멤버 메시지에 붙인 `http(s)` URL 이 다른 메신저처럼 클릭 가능한 링크가 되고, 말풍선 아래에 대표 이미지·제목·설명 **미리보기 카드**가 뜬다. 미리보기는 서버가 대신 열어 OG 메타를 읽어오며(로그인 계정만), 사내망·로컬 주소 차단(SSRF 방어)·리다이렉트 재검증·응답 크기 상한을 두고 Upstash 에 캐시(성공 7일/실패 1일)한다.
- **이미지·파일 첨부.** 대화방 하단 `+` 메뉴의 **"🖼 사진·파일"** 로 이미지·파일을 골라 메시지에 담아 보낸다(캡션과 함께, 최대 10개). 받은 메시지의 이미지는 그리드 썸네일(탭하면 원본), 파일은 다운로드 카드로 보인다. 저장은 **Vercel Blob**(메시지엔 URL 만, 서버가 Blob 호스트·용량 재검증). 업로드 토큰은 로그인 계정만 발급받고, blob 클라이언트 SDK 는 첨부를 실제로 고를 때만 로드한다. **Blob 스토어(`BLOB_READ_WRITE_TOKEN`) 연결 시에만 노출** — 미설정이면 버튼이 숨겨진다.
- **이메일/비밀번호 로그인.** Google 외에 이메일+비밀번호로 가입·로그인할 수 있다. finz 계정은 provider-agnostic 이라 기존 계정 모델(`finz_auth_links`)에 `provider="password"` 로 붙고, 로그인 후 흐름(핸들·캐릭터 온보딩 → 4탭)은 Google 과 동일하다. 비밀번호는 새 패키지 없이 Node 내장 `scrypt`(랜덤 salt + `timingSafeEqual`)로 해시하며 평문은 저장·로깅하지 않는다. 같은 이메일이라도 Google 계정과 자동 연결하지 않는다(미검증 → 계정 탈취 방지).

### Fixed
- **스레드 답글 라벨 정렬 통일 + 모바일 답글 폭 넘침.** 스레드 모드에서 원글 아래 답글 어포던스("답글 달기"/"💬 답글 N개")를 원글 발신자와 무관하게 **항상 오른쪽 정렬**로 통일했다. 또 긴 답글·URL 이 모바일에서 가로로 넘쳐 잘리던 문제를 말풍선 `min-width:0` + `overflow-wrap` 으로 고쳐 화면 폭 안에서 줄바꿈되게 했다.

## [0.1.32.0] - 2026-07-04

### Added
- **채팅방 스레드/리플라이(슬랙식) + 방별 대화 방식 선택.** 채팅방 설정(⚙️)의 "💬 대화 방식"에서 **일반 채팅 / 스레드** 를 고를 수 있다(기본 일반 — 기존 방·사용자 변화 없음). 스레드 모드에선 메인 타임라인에 원글만 보이고, 각 원글 아래 **"💬 답글 N개"** 가 뜬다. 탭하면 **전용 스레드 화면**(원글 + 답글 + 답글 입력창)이 열려 곁가지 대화를 정리해 이어갈 수 있다.
  - 스레드 안에서 **@finz** 를 부르면 그 스레드 맥락으로만 답하고, 답변도 그 스레드에 달린다.
  - 데이터: 새 저장 필드 없이 기존 답장(`replyTo`)으로 스레드를 그룹핑(append-only 유지). 방에는 `chatMode` 하나만 추가. 모드 전환은 **무손실**(되돌려도 대화 그대로)이며 토글이 폴링으로 전 멤버에게 ~3초 내 전파된다.
  - 방 대화 방식 변경은 방 멤버만(세션 인증 + members-guard). 순수 셀렉터 단위 테스트 + 실브라우저 렌더 검증.

## [0.1.31.5] - 2026-07-03

### Added
- **하단 + 메뉴에 '🤖 FINZ에게 물어보기'.** 누르면 입력창에 자동으로 `@finz ` 가 채워지고 커서가 놓여, 바로 질문을 이어 쓸 수 있다.

### Changed
- **finz 가 이제 @finz 멘션에만 응답한다(선제 개입 비활성).** 이전엔 멘션이 없어도 대화가 쌓이면 finz 가 스스로 끼어들어 LLM 을 썼다(토큰 소모 + 예기치 않은 개입). 이제 @finz(또는 + 메뉴 'FINZ에게 물어보기')로 부를 때만 답한다. 되살리기 쉬운 플래그(`FINZ_PROACTIVE_ENABLED`)로 관리.

### Fixed
- **finz 답변이 `{ "finz_message": ... }` 같은 날 JSON 으로 오던 버그 수정.** 원인: LLM 호출이 응답 스키마가 없어도 JSON 출력 모드로 강제돼, 모델이 필드명을 제멋대로 지어낸 JSON 을 뱉었다(선제 개입 경로). JSON 모드는 **응답 스키마가 있을 때만** 켜지도록 고쳐, 스키마 없는 자유 답변은 자연스러운 평문으로 온다. gemini-2.5-flash·flash-lite 실제 API 로 검증.

## [0.1.31.3] - 2026-06-29

### Fixed (보안·UX 데모 퀵윈 7건)
- **방 대화·포트폴리오를 멤버만 볼 수 있게(읽기 권한 가드).** 이전엔 대화방 링크(ID)만 알면 비멤버도 방의 대화·한줄입장·포트폴리오 보유내역을 읽을 수 있었다. 이제 메시지 조회(SSR 시드·폴링)와 그룹 정보 조회가 모두 로그인 멤버에게만 허용된다. (쓰기 라우트의 세션 검증은 후속 작업.)
- **@finz 가 생각하는 동안 '입력 중' 표시.** @finz 를 부른 직후 의도를 파악하는 ~1초 동안 아무 반응이 없어 봇이 멈춘 것처럼 보였다. 이제 부르는 순간부터 답이 올 때까지 타이핑 표시가 이어진다(차트·시황·포트폴리오 요청 포함).
- **빠르게 연속으로 보낸 메시지가 '전송 실패'로 뜨던 문제.** 짧은 간격으로 보내면 레이트리밋에 걸려 실패로 표시됐다. 이제 잠깐 뒤 자동으로 한 번 재전송해 메시지를 잃지 않는다(같은 id 라 중복 안 생김).
- **처음 들어온 친구의 막다른 길 제거.** 초대 링크로 들어왔는데 캐릭터가 없으면 합류가 막힌 채 빠져나갈 길이 없었다 — 이제 '캐릭터 만들러 가기' 버튼이 뜬다. 또한 캐릭터 없는 새 계정은 첫 화면을 빈 대화 탭 대신 프로필로 보낸다.
- **키보드 포커스 표시 추가.** 키보드(접근성)로 탐색할 때 현재 위치가 안 보이던 문제 — 코랄 윤곽으로 표시(WCAG 2.4.7, `--fz-*` 토큰만 사용).

### Internal
- **@finz 가 Gemini 무료 한도에 닿으면 잠깐 쉬도록(429 기록).** 한도 도달 후에도 매 호출이 무의미하게 429 를 반복하던 것을, 차단 상태를 KV 에 기록해 다음 호출을 건너뛰게 함(`recordLlmQuota` 헬퍼 + finz LLM 라우트 10개). "finz 가 계속 답을 못 함" 완화.
- 가장 빈번한 폴링 경로(chat GET)의 그룹·계정 조회를 병렬화해 cross-region(도쿄·싱가포르) 왕복 지연 누적 방지.
- 근거: `docs/finz/2026-06-29-finz-deep-review.md` (10개 관점 심층 리뷰)의 Quick wins 1~7.

## [0.1.31.1] - 2026-06-29

### Fixed

## [0.1.31.0] - 2026-06-28

### Added
- **채팅방별 알림 음소거 (+ 멘션 예외).** 채팅방 설정(⚙️)에 "🔔 알림" 섹션이 생겼다. **"이 방 알림 끄기"** 를 켜면 그 방의 새 메시지 푸시 알림을 안 받는다. 음소거해도 **"멘션은 받기"**(기본 켜짐)가 켜져 있으면 누가 나를 **@표시이름**으로 부를 때는 알림이 온다 — 카카오톡식 동작.

## [0.1.5.0] - 2026-04-24

### Changed
- `/travel` 결과 상단의 한 줄 `summary_line` 을 제거하고, **2~4문장짜리 `rationale` 필드**로 교체. 일정 설계 근거(왜 이 구성인지·사용자 요청 반영 방식) + 달성 못한 요청의 사유(특히 예산 초과 시 "요청 X만원 · 예상 Y만원 (Z만원 초과)" 숫자 명시 + 대안 제시) 를 담음. 이전 `summary_line` ("맞춤형 일정입니다" 류 boilerplate) 이 정보량이 없다는 피드백 반영.
- `TravelPlan.summary_line: string` → `TravelPlan.rationale: string` (schema 변경, 파서·UI·시스템 프롬프트 동기 수정).
- `/api/gemini` 의 `maxTokens` 4096 → 6144 상향. rationale 추가로 응답 토큰 여유 확보 (특히 3박 이상 + 긴 gap 설명 시 truncation 방지).

### UI
- `PlanCard` 헤더 본문 스타일: `text-base font-medium` → `text-sm leading-relaxed` (2~4문장 가독성), `whitespace-pre-wrap` 추가로 LLM 이 문단 구분 시 정상 렌더.
