# secondwind 킥오프 미팅 회의록

> 이 회의록은 참석자 중 한 명(akushi)의 기억을 바탕으로 AI 가 정리한 초안입니다. 다른 참석자 검토 후 누락·오류를 보완해주세요.

---

## 1. 모임의 최종 목표

참석자별로 원하는 지점이 달라서, 공통 교집합을 찾기보다 각자의 동기를 기록해두고 이후 로드맵에 반영하기로 함.

- **김태훈** — 앱스토어/플레이스토어에 우리 서비스를 올리고, 규모가 작더라도 수익화까지 경험해보기. 본인에게도 실제로 필요한 서비스를 만들고 싶음.
- **남덕우** — 큰 수익으로 이어질 수 있는 다양한 시도를 해보고 싶음.
- **김지헌** — AI 생태계를 지속적으로 경험하고, 본인과 지인들이 실제로 사용 가능한 서비스를 만들어본 뒤 그 이후의 목표를 설정해보고 싶음.

**공통 지점:** 세 사람 모두 *실제 사용자가 쓰는 형태까지* 가져가 보는 것에 동의. 수익화 수준·규모에 대한 기대치는 참석자마다 다름.

---

## 2. 비용이 발생하는 영역

- **개인별 AI 도구 사용 비용** — Claude Code, Codex 등 코딩 보조 도구 구독료.
- **공용 인프라 비용** — 서버, 데이터베이스, 서비스 내에서 이용할 AI LLM API 등.

---

## 3. 비용 감당 수준

### 개인별 AI 도구

- **각자 부담.**
- Claude Code Plus 요금제(월 20달러 수준)는 개인이 감당 가능한 수준으로 합의.
- 다만 초기에는 아이데이션·기획 중심이므로, 현재 이용 중인 **ChatGPT Plus 요금제에 Codex 를 연결해서 사용**하다가, 실제 코딩 단계로 넘어갈 때쯤 Claude Code 결제를 고려하기로 함.

### 공용 인프라

- **세 명 합쳐서 월 50달러 정도까지 감당 가능.**
- 한 명이 공용 비용을 대표로 결제한 뒤, 나중에 정산하는 방식.

---

## 4. 비용·복잡성에 대한 의견

- **외부 의존성이 늘어날수록 비용·복잡성이 급증**할 수 있음 — 지도 API, 외부 DB, LLM, 기타 외부 API 등.
- **Native 기능이 늘어날수록 구현 난이도가 올라감** — 카메라, GPS 등.
  - 최근 예시로 본 PWA 형태의 앱들은 단순 웹 서비스를 앱 형태로 패킹한 것이라, 단말기 네이티브 기능 활용이 제한적임.
- **초기에는 저장소·LLM 등이 필요 없는 최소 의존성 접근이 바람직**.
- 게시판처럼 공개된 데이터를 서로 공유해야 하는 경우가 아니라면,
  - 단말기 로컬에만 저장하고 **개인 구글 드라이브 등으로 동기화**하는 전략도 유효.

---

## 5. 로드맵에 대한 의견

프로덕트 로드맵뿐 아니라 **모임 자체의 로드맵**도 필요하다는 공감대.

예시 구분:

| 단계 | 목표 |
|---|---|
| Phase 1 | 상호 바이브 코딩이 가능한 앱 서비스 환경 구축 |
| Phase 2 | 지인 대상으로 사용성 확보 |
| Phase 3 | 앱스토어·플레이스토어 배포 |

**합의:** 다음 모임까지 각자 **Phase 1 프로덕트 아이데이션**을 해오기로 함.

## 6. Phase 1 아이데이션 후보

각자 다음 모임 전에 구체화해볼 후보 주제:

1. **여행 계획 서비스** — LLM 기반 장소·경로 자동 추천, 공유, 편집 기능
2. **투자 인사이트** — 구체 포맷 미정
3. **중간 지점 서비스** — 기존 MeetPick 개선 방향
4. **부부육아일기**
5. **영상을 블로그 형태로 추출**
