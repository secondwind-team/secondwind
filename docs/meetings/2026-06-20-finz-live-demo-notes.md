## 0. 시연 시작 — 캐릭터 소환·핸들·친구 추가

- 각자 취향 카드 선택 → **캐릭터 소환까지 완료**. (캐릭터 프로필에서 소환되게끔 흐름을 정비해 둠 — "소환됐다기보다 '내가 무슨 캐릭터다'가 나온 상태")
- **핸들 설정·친구 초대** 사용 (예: `더나인`/D9, `아쿠시` 등). 서로 친구 추가.
- **우정주는 대화방 안에서만** 가능하며 **친구가 최소 두 명 이상** 있어야 함. (1:1 대화로는 안 되고, 그룹이 만들어지면 시작하자마자 우정주가 뽑힘)

## 1. @핀즈 = AI 트리거 시연

- 채팅방에서 AI를 쓰는 트리거는 **`@핀즈`(골뱅이 핀즈) 멘션**. 예: `@핀즈 오늘 테슬라 주가 알려줘` → AI가 알아서 추가 검색해 답함.
- **실제 LLM이 연결돼 있고 토큰 비용이 청구되는 유료** 상태. (트래블/여행 만들 때처럼 토큰이 금방 소진되는 경험을 다들 공유) 다만 **텍스트 수준은 우리 정도 사용량에선 비용이 매우 적음** — "한참 대화해도 천 원 안 나올 것". 상용화 단계는 완전히 다른 얘기.
- **사용량 확인**: 오늘 기준 약 2.9원 사용(실시간 아니라 반영이 약간 늦음). 다만 현재 **세컨드윈드가 하나의 키로 묶여 있어** 트래블의 지도 검색 사용량과 합산됨 → **나중에 분리**해서 봐야 함.
- 우정주 뽑기 버튼은 있으나, **`@핀즈`로 직접 트리거하는 연결은 아직 미완(빌드 중)**.

---

## 2. 우정주 뽑기 동작 확인 — 종목 vs 테마

- 우정주는 **두 가지 형태**로 나옴: **종목을 찍어주기도 하고, 테마 형태로 나오기도** 함. (이번 시연에선 "구독형 콘텐츠 플랫폼"이라는 **테마**가 나옴 → 각자 입장 남기기 → AI 요약 흐름)
- 이 종목/테마 분기는 지헌이 의도적으로 넣었다기보다, **함께 만든 버전에서 자연스럽게 나온 동작**(AI가 "설계 의도"라고 답한 적 있음).
- **의견(태훈)**: 후속 대화를 이어가기에는 **테마보다 종목이 나오는 편이 더 낫다**.

## 3. 메신저 구조 확정 — 친구 목록 + 대화 목록 + 피드

- 메신저 형태로 개편. **피드 탭**에는 "○○가 우정주를 만들었어요 / 미래기술 딜러 캐릭터를 소환했어요" 식의 활동이 SNS처럼 노출됨.
- **탭 구성**: 친구 목록 + 대화 목록 두 개는 카카오톡·텔레그램 등 메신저 **공통**, **피드는 지헌이 임의로 추가**한 것. 카카오톡으로 치면 친구 목록의 "최근 업데이트/소식"이 피드에 해당. → 사용자가 가장 많이 쓰는 영역이므로 **아이디어를 모아 바꿔도 됨**.
- 메신저 자체는 대단한 기능이 아니지만, **이 큰 틀(기반) 위에 컨텐츠를 하나씩 붙여나가는 방식**으로 간다. (프로필 위치, 점점점 메뉴 등은 카톡 관습 참고)

> "메신저라는 기반·기록이 생겼으니, 여기서부터 컨텐츠를 생각해보고 붙일 수 있다."

논의: 결국 **대화방 안에서의 기능 / 서비스의 매력 포인트**를 고민해야 하는 단계. 메신저 형태가 잡히고 대화가 생기니 다음 아이템을 고민할 방향성이 보이기 시작.

## 4. PWA 모바일 설치 안내

- 모바일 확인 방법: ① 핸드폰 브라우저로 주소 직접 접속, ② 데스크탑 크롬 우클릭 → 검사 → 노트북/핸드폰 아이콘으로 **모바일 시뮬레이션**(아이폰 등 장치 선택).
- **설치형 PWA** 형태(트래블 만들 때 이미 PWA 설치 구현해 둠). 사파리: **공유 → 홈 화면에 추가**, 크롬: **우상단 메뉴 → 홈 화면에 추가/다운로드**.
- ⚠️ **preview 로그인 막힘**: `@핀즈`(finz)까지 진입하면 구글 로그인으로 넘어가는데, **"구글 정책을 준수하지 않는다"** 메시지가 떠 로그인 불가 → **세컨드윈드(secondwind)까지만 진입해서 설치**하면 됨. (preview 도메인이 OAuth 콜백에 미등록된 기존 이슈와 동일 맥락)
- 모바일 UI 버그 관찰: **iPhone 노치(카메라 부분)에 상단이 가려짐**, **채팅창을 누르면 자동 확대**되어 불편 → "짜치는 건 다 보정해야 함".
- PWA는 웹을 앱 형태로 포팅한 것이라 **기기 기능 접근이 제한적**. 알림은 가능하나 설정을 한 번 해줘야 함(아래 9번).

## 5. 아이디어 도출 — 브리핑 / 차트 / 집중 토론방

오늘 바로 해볼 만한 아이디어를 정리:

### 5-1. 정기 경제 시황 브리핑 (제안: 김태훈)

- 우정주로 정해진 종목에 대해, **24시간 최신 뉴스를 요약**(가격 / 호재·악재)해서 **매일 아침(8~9시) 채팅방에 리포팅**해주는 기능.
- 지헌 설명 — 이는 **AI의 선제적 트리거**. 현재 메신저의 AI는 **트리거 기반**(부르면 응답)인데, 스스로 트리거를 발동하려면 **스케줄러**를 달면 됨.
  - **능동적 탐색**(계속 뉴스를 검색하다 발견 시 발송)은 토큰 비용이 큼.
  - **정해진 시간 1회 발송**(예: 9시에 한 번만 검색)은 비용이 거의 안 듦 → 이 방식이 적합.
- 기대 효과: **출근길/퇴근길** 등 특정 시간대에 새로운 대화가 이어질 계기가 됨.

### 5-2. 차트 임베딩·고정 표시 (제안: 남덕우)

- 우정주 종목이 정해지면, **현재 차트/가격**을 채팅방에 **팝업처럼** 또는 **화면 상단에 고정**으로 보여주기. 스크롤과 무관하게 항상 보이고, 필요 시 눌러서 분봉/일봉 수준으로 확인.
- 지헌: LLM은 **텍스트로만 결과**를 주지만, **차트 임베딩으로 가져오는 건 가능할 것 같고 필요하다고 봄** → 즉석에서 시도(6번).
- **고정 표시 방법 2가지**:
  1. **카톡 공지사항 형태** — 대화 하나를 상단 고정. 단 "선점 방식"이라 다른 사람이 낚아채 바꿔치기 가능.
  2. **채팅방 설정에서 컨텐츠 유형별 분류**(예: "차트만 보기") — 현재 채팅방에 **설정 기능 자체가 아직 없음**(카톡 우측 메뉴의 이미지만 보기/파일만 보기 류).

### 5-3. LLM 지침(가이드) 미정비 — 메모

- 현재 **LLM에 지침이 하나도 없어 "날것의" 상태**. 그래서 "추천해줘" 같은 요청에 **"그런 거 추천 못해"라고 거절**하는 현상 발생.
- 대화방에서 AI가 **어떻게 탐색하고 어떻게 답할지** 가이드를 정비해야 함 → 이번엔 메모 차원으로만 기록(고도화 과제).

## 6. 트레이딩뷰 차트 임베딩 — 즉석 구현 ✅

- "바로 해볼 수 있는 게 트레이딩뷰 차트 임베딩"이라며 **회의 중 AI 에이전트(Claude)에 구현을 시키고 배포**까지 진행.
- 배포 후 새로고침하여 확인:
  - ✅ `@핀즈 테슬라 주가 차트 보여줘` → **트레이딩뷰 차트 임베딩 표시**, 누르면 링크까지 연결.
  - ✅ `비트코인` 차트도 동작 (원화 기준 표시).
- **API 키 불필요한 공개형 API** 사용 → 내부적으로 트레이딩뷰 임베디드 API. 바이낸스 등도 키 없이 쓸 수 있는 공개형 API를 제공. **인프라 추가가 없어서 좋음**(주가/암호화폐 정보 공개 API가 꽤 많음).

> "인프라가 추가 안 돼서 다행. 인프라 추가될 때마다 스트레스를 받는다."

## 7. 정기 브리핑 스케줄러 구현 시도

태훈의 브리핑 아이디어를 그 자리에서 구현 시도. AI에 다음 취지로 요청:

> "특정 시간에 특정 프롬프트를 수행한 결과를 채팅방에 보내는 기능을 만들어줘. 예를 들어 **매일 오전 9시에 당일 경제 시황을 300자 내로 정리하고 주요 종목을 같이 작성해서 채팅방에 보내주는** 기능."

### 7-1. 비용·모드 메모

- 지헌은 현재 **Claude의 가장 비싼 모드(맥스 effort)** 사용 — 제일 비싼 LLM + 그 안의 effort 모드 중 max. **회사 요금제로 대략 월 $100 요금제 상당**. 토큰은 **5시간마다 초기화**되어, 아까우니까 여러 주제를 계속 던져두고 돌리는 식으로 활용.

### 7-2. 스케줄러 방식 결정 → GitHub Actions

- **cron(크론/크론탭)** = 요일·날짜·시각 등 반복 설정에 따라 이벤트를 실행하는 유닉스 표준.
- 후보 비교:
  - **Vercel(버셀) Hobby(무료) 플랜의 cron** — **하루 1회만 가능**. 우리끼리 9시 1회만 쓸 거면 충분하지만 확장성 제약. (참고: secondwind 인프라가 대부분 Vercel에 들어가 있고 무료로 쓰는 중인데, 최근 한도에 자주 걸림 — 별도 논의 예정)
  - **GitHub Actions** — **무료, 실행 제한 없음, 이미 사용 중, Actions 래퍼라 자연스러움**. 단점은 **best effort라 정시가 아니라 몇 분 지연**될 수 있음.
- **결정: GitHub Actions 채택.** 인프라 추가 없이 가능. (정각 쏠림을 분산하는 "스프레드" 개념 때문에 정시 보장은 안 되지만 브리핑 용도엔 무방)
- 스케줄 조건/트리거 방식을 관리할 **별도 관리 화면**이 필요할 수 있음(ChatGPT 스케줄 기능처럼).

### 7-3. (예상 밖) 구독형 컨텐츠 발견

- AI가 MVP를 만들면서, 지헌이 의도한 형태("그냥 활동 방 전체에 발송")가 아니라 **"경제 시황 컨텐츠를 미리 만들어 두고, 핀즈에서 그 컨텐츠를 구독"** 하는 형태로 구현 → **의도와 달랐지만 좋은 아이디어**로 이어짐.
  - 발송 대상 설정 필요: **"활동 방 전체"** vs **"구독한 방만"**. 사람이 많아지면 모든 방에 보내면 안 되므로 결국 **설정·구독 개념이 필요**(앞서 말한 채팅방 설정의 필요성과 연결).
  - 임시 구독 방 이름 예: **"매일 아침 시황 받을래"**.
- **유료화 가능성(덕우·태훈)**: 수만 개의 방이 생겨도 주요 종목(예: 비트코인)의 차트 등 **누구나 제너럴하게 쓸 컨텐츠를 패키징**해 **방 단위로 구독**시킬 수 있음 → 서비스 출시 시 유료화 포인트.

---

## 8. 집중 토론방 vs 자유 대화방 (제기: 김태훈)

- 태훈의 처음 구상: **우정주로 종목 하나가 정해지면, 그 종목을 중심으로 대화·기능이 종속**되는 형태. (예: 우정주가 SpaceX면 → 차트도 SpaceX만, 구독도 SpaceX만. 방에서 우정주를 바꿀 수는 있음)
- **트레이드오프**:
  - 너무 자유롭게 풀면 → 대화가 옆으로 새고 산만해질 가능성↑ (우리끼리도 시선이 계속 분산됐던 경험).
  - 너무 조이면 → 자유로운 사고가 막힐 수 있음.
- **지헌의 정리**: **제약의 중요성에 공감**. 다만 둘 중 택일이 아니라, **"집중 토론"을 하나의 새로운 컨텐츠 형태로 분리**하는 방향 제안. 산발적 자유 대화방으로 흘러가다가 "이 종목으로 집중 토론 해볼까?" 하면 **별도의 집중 토론방**이 생기고, 그 방에선 **제한된 범위의 기능만** 쓰게 하는 식.
- 단, **아직 채팅을 어떻게 끌어갈지 모르는 실험 단계**이므로, 일단 이것저것 써보며 인사이트를 모으기로.

---

## 9. PWA 알림 설정 안내 (안드로이드)

- **알림 기능은 아직 미구현** → 지헌이 중간에 알림 기능을 추가한 뒤 **별도로 설치·설정 가이드를 다시 안내**하기로.
- 안드로이드(갤럭시) 알림 설정이 까다로움 (커플렛 만들 때 경험):
  1. **세컨드윈드 앱**: 배터리 최적화 안 함(제한 없음) — 최적화하면 알림 서비스를 OS가 자동으로 꺼버림.
  2. **구글 플레이 서비스**: 배터리 제한 없음.
  3. **알림 허용 + 토스트 팝업 스타일** 설정.
- **iOS는 별 설정 없이** 토스트 팝업이 바로 옴(아이패드·갤럭시 테스트 결과 오히려 iOS가 수월). 배터리 광탈 등 부작용은 없음.

---

## 10. 기여 방식·인프라 원칙

- **전원, 평소 짬짬이 개별 피처를 만들어 PR 제출** → 상황에 따라 바로 반영. **기존 구조에 붙이는 형태**라 작업 범위가 한정적이어서 작업 가능성이 높음. 미팅 때가 아니어도 기여 가능.
- **인프라(API 키 등) 추가는 별개 문제**:
  - 현재 계정이 거의 다 **지헌의 계정**에 묶여 있음. **새 계정을 파면 관리가 난잡**해짐 → API가 필요한 피처는 **개발 전부터 사전 협의** 필요.
  - 다행히 이번 트레이딩뷰는 **공개 API(키 불필요)**라 인프라 추가 없이 구현됨.

---

## 11. 잡담 — 주사모와 실사용

- 굳이 억지로 컨텐츠를 끄집어내기보다, **주사모(원래 투자 대화 모임)에서 하던 일상적·가벼운 대화를 핀즈에 쌓아볼 필요**가 있음 → 대화가 쌓이면 **AI가 우리 대화를 분석**(각자 특징 파악)할 수 있게 됨. ("오늘 암호화폐 왜 폭락?" 같은 가벼운 대화부터 메신저에서 해보자)
- 주사모 근황: 국장/미장/코인 등 **서로 관심 주제가 달라** 대화가 길게 이어지진 않지만, 그만큼 **건전하고 넓은 채널**. 특별한 뉴스가 있을 때 짧게 시작되는 추세.

## 주요 결정사항

- **플랫폼 구조**: **메신저 형태로 확정** — 친구 목록 + 대화 목록 + 피드 탭. (피드 탭은 임의 지정, 아이디어 모아 변경 가능)
- **AI 트리거 방식**: **`@핀즈` 멘션 시 LLM 호출**. 현재는 지침 없는 날것 상태 → 추후 **프롬프트 가이드 정비 필요**.
- **트레이딩뷰 차트 임베딩**: **API 키 불필요한 공개형 API** 활용, **즉석 구현 완료**(테슬라·비트코인).
- **스케줄러 구현 방향**: Vercel 무료(Hobby) 플랜 cron 하루 1회 제한 때문에 **GitHub Actions 방식 채택**.
- **인프라 원칙**: **인프라 추가 없이 기존 구조에 기능을 붙이는 방식 유지**. API 키가 필요한 피처는 개발 전부터 사전 협의.
