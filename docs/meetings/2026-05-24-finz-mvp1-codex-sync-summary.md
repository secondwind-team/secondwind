# 2026-05-24 FINZ MVP 1 Codex 구현 싱크 — Zoom AI 요약

> Codex로 FINZ MVP 1단계(취향 카드 선택 → 캐릭터 소환) 구현 진행 상황을 공유하고, 비개발자 팀원과 AI 개발 도구 사용법을 익힌 회의입니다.

## 핵심 성과

Codex를 활용하여 FINZ MVP 1단계의 UI(취향 카드 선택 및 캐릭터 소환 화면)를 성공적으로 구현했고, AI 기반 개발 도구 사용법을 팀원들과 공유했다. 기획 문서를 바탕으로 자동 생성된 MVP 계획서와 최소 데이터 모델을 함께 검토하며 프로젝트 방향성을 다시 확인했다.

## 확정된 결정사항

- **서비스 이름**: FINZ로 최종 확정 (2026-05-10 네이밍 회의에서 결정된 명칭 재확인)
- **MVP 1 범위**: 취향 카드 선택 → 개인투자 캐릭터 소환 → 그룹 생성 진입까지 구현
- **초기 로그인 제외**: MVP 1에서는 로그인 기능을 의도적으로 미루고 로컬 스토리지 기반 개인 환경에서만 동작
- **데이터 저장 방식**: 로컬 스토리지를 사용하여 새로고침 시에도 데이터 유지
- **디자인 시스템**: 현재는 SecondWind 디자인 시스템을 따르되, 추후 FINZ에 맞는 디자인으로 교체 예정

## 기술 구현 현황

- **개발 환경**: Monorepo 구조 — 여러 앱을 하나의 폴더에서 동시 개발 가능하도록 구성
- **사용 도구**: Codex Desktop App을 통해 자연어 기반 코드 생성 및 Git 명령 실행
- **프로젝트 구조**: `/app` 폴더에 모든 소스코드, `/docs` 폴더에 문서 관리
- **로컬 서버**: `localhost:3000`에서 개발 서버 실행 중
- **패키지 관리**: npm(Node Package Manager)을 통한 라이브러리 설치 및 관리

## MVP 계획서 주요 내용

### 제품 핵심 개념

- 투자 수익률 경쟁이 아닌, **투자 취향을 캐릭터처럼 이해**하여 우정주 중심의 대화를 생성하는 게임형 대화 앱
- 증권사가 아닌 게임에 가깝게 — 종목 정답성보다 **대화 가능성** 우선
- 실제 금융, 돈, 자산은 다루지 않음

### MVP 1 구현 화면

- 취향 카드 선택 시작 화면 (이름 노출, 짧은 설명, 카드 선택)
- 개인투자 캐릭터 소환 (클래스, 레벨, 타이틀, 한 줄 설명, 스탯, 약점)
- 친구 그룹 생성 (그룹 이름, 캐릭터 파티 추가, 친구 초대)
- 오늘의 우정주 제안 (AI 기반 종목 제안)
- 투자 레이드 (역할 기반 대화, 포지션 입력)

### 최소 데이터 모델

| 엔티티 | 필드 |
|---|---|
| FINZ Taste Card | `id`, `label`, `tags` (배열) |
| FINZ Character Card | `classId`, `className`, `level`, `title`, `summary`, `stats` (객체), `weakness`, `tips` |
| FINZ Member | `id`, `name`, `characterId`, `createdAt` |
| FINZ Group | `id`, `name`, `members` (배열), `createdAt` |

## 보류 및 검토 사항

- **캐릭터 톤 적합성**: 게임 캐릭터 느낌이 대화방 개념과 잘 조화될지 검토 필요
- **레이드 기능의 지속 가능성**: 우정주 기반 대화가 얼마나 지속 가능할지 확신 부족
- **MVP 화면 수**: MVP 치고 화면이 많은 편이라는 의견
- **디자인 참고 사례**: FINZ 컨셉에 맞는 디자인 시스템 참고 사례 조사 필요

## 기술 학습 내용

- **Codex 사용법**: `@` 기호로 문서 참조, `!` 기호로 터미널 명령 실행, 자연어로 Git 작업 수행
- **모노레포 구조**: 하나의 저장소에서 여러 프로젝트를 관리하는 방식
- **브랜치 전략**: main 브랜치는 직접 커밋 금지, feature 브랜치로 작업 후 MR(Merge Request) 방식
- **반응형 개발 도구**: 브라우저 개발자 도구에서 모바일 화면 시뮬레이션 가능
- **스킬(Skill) 개념**: 자주 사용하는 프롬프트를 모아놓은 명령 집합

## 기술적 이슈
