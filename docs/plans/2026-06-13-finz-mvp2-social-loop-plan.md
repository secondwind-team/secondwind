## 0. 이 문서가 답하는 질문

1. **왜 지금 흐름이 어색한가** (실사용자 관점 진단)
2. **FINZ의 큰 흐름(북극성)은 무엇인가** — 여러 기능을 하나의 제품으로 묶는 through-line
3. **단기 MVP는 무엇을 만들어야 하는가** — 다음 작업 단위
4. **코드를 더 쓰기 전에 닫아야 할 결정은 무엇인가**

핵심 결론 한 줄:

> **지금 배포된 FINZ는 "친구 없는 우정주 앱"이다.** 제품의 모든 카피·기획은 *친구·파티·공유*를 약속하는데, 코드는 끝까지 **혼자 카드 고르고 → 혼자 캐릭터 보고 → 혼자 우정주 한 장 받고 끝나는** 1인 경험이다. 그 1인 결과물조차 Google 로그인 뒤에 숨어 있다. 다음 작업은 새 기능 추가가 아니라 **빠진 사회적 절반(두 번째 사람)을 최소 단위로 채우는 것**이어야 한다.

## 1. 현재 배포 현실 (2026-06-13 기준)

코드로 확인된 사실. `app/(site)/finz/` 전체에 그룹 생성·두 번째 멤버·파티·레이드·공유 버튼·초대 링크가 **하나도 없다.**

**실제 동작하는 것 (`live`):**

- `/finz` 단일 화면 — 취향 카드 20종 그리드 (`app/(site)/finz/page.tsx`, `_components/finz-taste-selector.tsx`)
- 취향 카드 3개+ 선택 → **deterministic** 캐릭터 소환 8종. AI·네트워크·인증 불필요 (`lib/common/services/finz.ts:273-290`, 단위 테스트 있음)
- Google 로그인(NextAuth) → 프로필을 **Neon Postgres**에 저장 (`app/api/finz/profile/route.ts`, `lib/server/finz-store.ts`)
- 로그인 사용자 1인의 "오늘의 우정주" Gemini 생성 + 일자별 캐시 + 재생성(force) (`app/api/finz/pick/route.ts`)
- 안전 문구("투자 조언이 아니라 대화 소재") 반복 노출

**기획했으나 없는 것 (`missing`):**

- 친구 그룹 / 파티 / 두 번째 멤버 — 엔티티·화면·API 전무
- `/finz/group`, `/finz/raid` 화면
- `POST /api/finz/facilitator` (AI 진행자 대화 루프)
- 투자 레이드: 보스·역할별 미션·한 줄 포지션·파티 요약
- 공유/초대 메커니즘 (UI에 share/tease 카피만 있고 동작 0)
- AI 실패 시 deterministic fallback 픽 (캐릭터만 fallback 있고 픽엔 없음)

### 기획 vs 현실 차이표

| 항목 | 기획 (05-09/05-24 plan) | 실제 코드 (현재) |
|---|---|---|
| 핵심 정체성 | 친구 **파티 조합** 기반 우정주 (멀티) | **1인** 취향 기반 우정주 (솔로) |
| 로그인 | "의도적으로 미룸" | **Google OAuth 필수** (저장·생성 모두 401) |
| 저장소 | "DB 없이 localStorage" | **Neon Postgres** (user_email 키) |
| 화면 | `/finz`, `/group`, `/raid` 3개 | `/finz` **1개** |
| AI 진행자 | `/api/finz/facilitator` 루프 | **미구현** |
| 레이드/포지션 | 보스·미션·포지션·요약 | **전부 없음** |
| 우정주 형태 | `whyThisParty[]`, 멤버별 `rolePrompts[]` | 단일 `whyThisFits`, `rolePrompt` 1개 |
| AI 실패 시 | deterministic fallback 약속 | fallback 없음 (에러만 반환) |

**plan과 code가 서로 다른 제품을 기술하고 있다.** 이 문서가 그 간극을 닫는다.

## 2. 진단 — 왜 흐름이 어색한가

### (1) 친구 없는 우정주 (싱글플레이어 함정)

비전의 핵심 가설 3은 *"개인 캐릭터만으로 끝나면 싱글플레이가 되므로, 반드시 그룹의 공통 콘텐츠로 이어져야 한다"*였다. 그런데 구현은 **정확히 그 함정에 빠진 버전**을 출시했다. 캐릭터 화면은 "친구에게 공유할 한 줄", "다음 레이드 역할"을 약속하지만 공유할 대상도, 갈 레이드도 없다 — 약속만 있고 동작이 없는 **유물(vestigial) 카피**가 어색함을 증폭한다.

### (2) 정체성 결정의 정면 번복 (05-31 → 06-06)

- **05-31**: "AI가 **파티 조합** 기반으로 우정주 자동 픽" — 멀티플레이가 V0의 핵심.
- **06-06** (개발환경 세팅 회의): 범위가 "로그인 → 취향카드 → 캐릭터 → AI 우정주 → 저장"으로 재정의되며 **"파티"가 조용히 사라지고** Google 로그인 + Neon DB가 들어왔다. Codex는 "로그인·DB 보류하고 혼자 쓰는 최소본부터"를 권했지만 팀이 "풀 사이클 완성"으로 뒤집음.

즉 **인프라 세팅의 편의가 제품 정체성을 재정의해버렸다.** 풀 사이클을 만들려다 가장 어려운 "사회적 절반"을 빼고 만만한 "로그인+DB+1인"으로 대체한 것.

### (3) "여러 명의 요리사" 봉합선

회의록을 추적하면 **최소 5개 화자**(지헌·태훈·덕우 + Codex/G-Stack + 회의록 요약 AI)가 제품을 양방향으로 당겼고, 어느 축도 닫히지 않았다:

- 실전형(후보 중 선택) ↔ 토론형(AI 자동 픽)
- 게임성(레이드·보스·스탯) ↔ 대화/공감 커뮤니티 — 매 회의 진동(05-09 게임 방향 → 05-24·05-31 "너무 유치한가, 우리가 하려던 게 이게 아닌데")
- 미니멀(Codex) ↔ 풀스펙(팀)
- 로컬 우선 ↔ 로그인+DB

이 미해결 긴장이 코드에 화석으로 남았다. 그리고 **구현이 plan보다 앞서 달리면서** plan과 code가 어긋났다.

### (4) 변수(variance) 부재

지헌의 05-31 지적: 같은 카드 → deterministic 같은 캐릭터 → 같은 우정주. 변수의 진짜 엔진은 *친구들의 포지션·반박·누적 대화*인데 그게 빠져 있으니, 1인 픽은 "매일 새로고침하는 운세" 수준의 얕은 재방문 동기만 남는다. **사회적 루프 자체가 변수 해결책이다.**

## 3. 큰 흐름 (북극성) — through-line은 "대화 한 판"

이 제품을 *하나로* 묶는 실은 캐릭터도 종목도 아니라 **"오늘의 가벼운 투자 수다 한 판"**이다. 지배 원칙이 *"사세요가 아니라 얘기해보세요"*인데 출시본엔 누군가와 얘기할 표면이 0개다. 모든 기능은 **"대화를 시작·지속·기억시키는 도구"**로 종속되어야 한다.

```
① 초대 링크로 입장 (로그인 없이)  →  ② 취향카드 3개+ → 캐릭터 즉시 소환
        ↓
③ 친구들 캐릭터가 RPG 파티로 모임  ("우리 파티 이렇게 나왔다" = 첫 공유 폭발점)
        ↓
④ AI가 파티 조합 + 누적 대화로 '오늘의 우정주' 1개 개봉
   (이유 + 싸울 포인트 + 캐릭터별 역할)
        ↓
⑤ 각자 '한 줄 포지션' 입력 → AI 진행자가 빈자리/반박을 받아치며 대화를 굴림
        ↓
⑥ 캐릭터 성장 (대화 활동으로 레벨업, 수익률 아님)  →  ⑦ 주간 파티 리포트 (wrapped 모먼트)
```

확장(예측카드·투표·공동 모의투자)은 ⑥⑦ 검증 후, 그리고 **항상 "누가 벌었나"가 아니라 "우리 판단이 어땠나"** 프레임으로만 붙인다.

## 4. MVP 2 범위 — 2인 사회적 루프

캐릭터 소환은 이미 잘 동작한다. 다음 한 걸음은 **빠진 사회적 절반의 최소 단위**를 채우는 것.

**MVP 2 한 문장:**

> 친구 1명을 링크로 초대해 두 캐릭터가 한 파티를 이루고, AI가 그 파티에 맞춰 연 우정주 하나에 둘이 각자 한 줄 포지션을 남기면 AI 진행자가 한 번 받아치는, **"둘이서 5분 수다 한 판"이 끝까지 도는 루프.**

### 유저 여정

1. A가 취향 카드 3개+를 골라 캐릭터를 소환한다 (이미 동작 / **로그인 불필요로 되돌림**).
2. A가 "친구 초대" 링크를 생성해 B에게 보낸다 (그룹 id를 URL/KV에 실어 복원, 실시간 인프라 없음).
3. B가 링크로 들어와 자기 캐릭터를 소환하면 두 캐릭터가 한 파티 화면에 모인다 — **"혼자가 아님"이 시각적으로 증명되는 결정적 순간.**
4. 누구든 "오늘의 우정주 열기"를 누르면 AI가 2인 조합 기반으로 우정주 1개 + 싸울 포인트 + 캐릭터별 역할 한 줄을 생성한다 (기존 단일 픽을 party shape로 확장).
5. A·B가 각자 한 줄 포지션(6종 stance + 코멘트)을 남긴다.
6. AI 진행자가 두 포지션을 읽고 **단 한 번** 받아친다 (안 한 사람에게 질문하거나 직전 의견에 리스크를 던짐 — 1-shot이면 충분, 무한 루프 불필요).
7. 결과 하단에 안전 문구 반복 + 파티 요약 한 줄로 마무리.

### 해야 할 것 (작게)

- **공유/초대 링크 1개** — 그룹 id 기반 복원. 실시간 동기화·푸시 없음. 비동기 방문으로 충분. *(현재 바이럴 엔진이 0)*
- **로그인 필수 → 선택** — 캐릭터·파티·우정주·포지션까지 무로그인 동작, 저장하고 싶을 때만 로그인 권유. (06-06 결정 되돌림이지만 팀의 "preview 공개로 친구 마찰 최소화" 가치와 정합)
- **pick을 party shape로 확장** — `whyThisParty[]`, 멤버별 `rolePrompts[{memberName, role, prompt}]`, `debatePoint`. **2명만 지원해도 사회적 가설은 증명된다.**
- **한 줄 포지션 입력 + 파티 요약** (자유 채팅 말고 구조화 입력 먼저).
- **facilitator 1회 응답** — 토큰 복리 비용/파티 해산 문제를 MVP에서 풀려 하지 말 것.
- **deterministic fallback 픽 + V0는 theme 기본** — Gemini 장애(06-06 미동작 이력)와 실명 종목 환각을 한 번에 방어.

### 명시적으로 자르는 것 (defer)

레이드 보스 전용 화면(`/finz/raid`), 캐릭터 성장/레벨/타이틀, 주간 리포트, 예측카드/투표/공동 모의투자, 캐릭터 AI 문장 다듬기, 아바타 이미지, 3명+ 실시간 동기화, FINZ 전용 디자인 시스템 교체. — 전부 *2인 루프가 재밌다고 확인된 뒤*.

> 특히 **공동 모의투자**는 "수익률 경쟁 앱"으로 변질될 최대 위험이므로 thesis가 단단해지기 전엔 금지.

## 6. 즉시 고칠 저비용 버그 (dogfooding 신뢰도 직결)

- **fallback 픽 부재** — `app/api/finz/pick/route.ts`는 Gemini 실패 시 에러만 반환(`:87-114`). plan이 약속한 deterministic fallback이 없어 핵심 보상이 통째로 죽음.
- **`UNIQUE(user_email, pick_date)` vs `profile_key` 불일치** — `lib/server/finz-store.ts`. 카드 바꿔 재생성하면 그날 단일 행을 덮어쓰고, 옛 profileKey로 GET하면 빈 응답 → "아까 그거 없어졌다"류 조용한 데이터 클로버링.
- **로그인 벽 mid-flow** — 비로그인 친구가 "오늘의 우정주 생성"을 누르면 말없이 OAuth로 튕김 (`finz-taste-selector.tsx:150-153`).
- **자잘한 것**: "N / 3 선택" 카운터가 목표 3처럼 오해됨(실은 최소); `generateDailyPick`의 자동 force-save 부작용(`:159`); 저장된 `character` JSONB 컬럼이 다시 읽히지 않는 dead data(`finz-store.ts:94-96`).

4. **`FINZ-MVP-05` 한 줄 포지션 + 파티 요약**.
5. **`FINZ-MVP-06` 2인 dogfooding 체크리스트**.

각 단계는 Feature ID 하나 기준의 작은 PR (PROJECT.md 원칙).

## 8. 성공 기준

수익률·종목 정확도가 아니다. **사회적 버전:**

- 친구 2명이 같은 파티에 들어온다.
- 둘 다 한 줄 포지션을 남긴다.
- 최소 1명이 "내일/다음 주제도 해보자" 반응을 보인다.
- 공유율이 0이면 wedge가 틀린 것 — 공유 가능한 산출물(캐릭터/파티 카드)이 바이럴 엔진이다.
