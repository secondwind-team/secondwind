## 한 줄 정의

FINZ는 친구들의 투자 취향을 게임 캐릭터처럼 소환하고, AI가 그 조합에 맞는 "오늘의 우정주"와 "투자 레이드"를 열어 대화가 시작되게 하는 앱이다.

## 제품 의도

FINZ의 목적은 투자 수익률 경쟁이 아니라, 친구들이 투자라는 맥락을 가볍고 즐겁게 이야기하게 만드는 것이다.

따라서 MVP 1은 증권앱처럼 시작하지 않는다. 첫 화면에서 종목 검색, 차트, 수익률 인증을 요구하지 않고, 넷플릭스 취향 선택처럼 부담 낮은 카드 선택으로 시작한다.

## 핵심 가설

1. 투자 지식이 적은 사용자도 추상 취향 카드는 고를 수 있다.
2. 취향 분석 결과가 리포트가 아니라 게임 캐릭터처럼 나오면 공유하고 싶어진다.
3. 개인 캐릭터만 있으면 싱글플레이가 되므로, 그룹의 공통 콘텐츠로 이어져야 한다.
4. "오늘의 우정주"는 매수 추천이 아니라 대화 소재일 때 FINZ의 톤과 맞다.
5. MVP 1 이후 투표, 예측 카드, 공동 모의투자로 확장할 수 있어야 한다.

## MVP 1 범위

MVP 1은 아래 4단계까지 만든다. 2026-05-31 범위 재정의: **AI가 파티 기반 "오늘의 우정주"를 뽑고, 그 우정주를 두고 대화가 이어지도록 질문/소재를 던지는 것까지를 완전한 V0 MVP로 본다.**

| 단계 | 기능 | 목표 Feature ID |
|---|---|---|
| 1 | 추상 취향 카드 선택 | `FINZ-ONBOARD-01` — 구현됨 |
| 2 | 개인 투자 캐릭터 소환 | `FINZ-CHARACTER-01` — 구현됨 |
| 3 | 친구 그룹 생성 또는 초대 | `FINZ-GROUP-01` — 샘플 친구 기반 v0 구현됨 |
| 4 | 그룹 조합 기반 AI 오늘의 우정주 선정 + 대화 진행 질문 | `FINZ-PICK-01` |

`FINZ-RAID-01` 의 실시간 채팅/투표/결과 복기는 MVP 1 이후 확장으로 둔다. 단, V0의 `FINZ-PICK-01` 은 AI가 대화를 시작하고 이어갈 질문까지 제공해야 한다.

### 1. 추상 취향 카드 선택

사용자는 실제 보유 종목이나 관심 종목을 입력하지 않고, 추상적인 투자 취향 카드를 3개 이상 선택한다.

예시 카드:

- 세상을 바꾸는 기술이 좋다
- 오래 버틸 수 있는 회사가 좋다
- 내가 직접 쓰는 브랜드가 좋다
- 한 방이 있어야 재밌다
- 현금흐름이 마음을 편하게 한다
- 사람들이 떠드는 곳에 기회가 있다
- 위기 때 줍는 게 좋다
- 남들이 놓친 저평가를 찾고 싶다
- 숫자보다 스토리에 끌린다
- 유행이 되기 전 먼저 발견하고 싶다

### 2. 개인 투자 캐릭터 소환

선택 결과를 바탕으로 AI가 개인 투자 캐릭터를 생성한다. 결과는 "안정형 투자자" 같은 정적 리포트가 아니라 게임 캐릭터가 등장하는 느낌이어야 한다.

캐릭터 결과 구성:

- 클래스명
- Lv.1 타이틀
- 한 줄 설명
- 스탯: 공격력, 방어력, 인내력, 정보탐색력, FOMO 위험
- 약점
- 친구에게 공유하고 싶은 한 줄 놀림

예시 캐릭터:

- 미래기술 딜러
- 배당 힐러
- 가치 탱커
- 브랜드 레인저
- 밈 버서커
- 매크로 마법사
- 위기 줍줍러
- 소비트렌드 정찰병

### 3. 친구 그룹 연결

개인 캐릭터는 그룹으로 이어져야 한다. 친구들이 합류하면 각자의 캐릭터가 파티처럼 보이고, 그룹 조합이 오늘의 우정주 추천의 입력이 된다.

예시 그룹 요약:

- 딜러 2명, 힐러 1명, 탱커 0명
- 상승장 대화력 S, 하락장 생존력 C
- 공통 관심: AI, 브랜드, 콘텐츠
- 약점: 방어 담당 부재

### 4. 오늘의 우정주

AI가 그룹의 캐릭터 조합과 취향을 바탕으로 오늘 이야기하면 재밌을 종목 또는 테마 하나를 제안한다.

원칙:

- "사세요"가 아니라 "얘기해보세요"다.
- 수익률보다 대화 가능성이 중요하다.
- 추천 사유는 그룹 캐릭터 조합과 연결되어야 한다.
- 실제 종목으로 시작할지, 섹터/테마부터 시작할지는 구현 전 추가 결정이 필요하다.

예시:

```text
오늘의 우정주: 디즈니
이 파티가 좋아할 이유: 브랜드 레인저와 콘텐츠 딜러가 동시에 반응할 만한 종목입니다.
싸울 포인트: 스트리밍은 회복 중인가, 이미 늦었나?
```

## AI V0 설계

AI는 FINZ MVP의 핵심이다. V0도 룰 기반 추천으로 대체하지 않고, 기존 secondwind의 Gemini 호출 구조(`callLlm`)를 재사용해 **파티 조합을 해석한 오늘의 우정주**와 **대화를 이어갈 진행 질문/소재**를 생성한다.

### 사용자 플로우

1. 사용자가 취향 카드를 고르고 개인 캐릭터를 만든다.
2. 친구가 합류하거나, V0에서는 샘플 친구를 선택해 파티 조합을 만든다.
3. 사용자가 "오늘의 우정주 뽑기"를 누른다.
4. 클라이언트가 파티 멤버의 캐릭터, 스탯, 태그를 `POST /api/finz/pick` 으로 보낸다.
5. 서버가 Gemini를 호출해 JSON 형태의 `FinzPick` 을 받는다.
6. 화면에 오늘의 우정주, 추천 이유, 싸울 포인트, 캐릭터별 관점, AI 진행 질문을 표시한다.
7. 사용자가 한 줄 의견을 남기면 `POST /api/finz/facilitator` 로 현재 대화 맥락을 보내고, AI가 다음 질문/반박 소재/조용한 멤버에게 던질 질문을 반환한다.

### 백엔드 경계

신규 API:

```ts
POST /api/finz/pick
POST /api/finz/facilitator
```

요청:

```ts
type FinzPickRequest = {
  party: {
    members: Array<{
      name: string;
      role: string;
      title: string;
      stats: {
        attack: number;
        defense: number;
        patience: number;
        research: number;
        fomo: number;
      };
      tags: string[];
      weakness?: string;
    }>;
    summary?: {
      headline: string;
      note: string;
      tags: string[];
    };
  };
};
```

응답:

```ts
type FinzPickResponse =
  | {
      status: "ok";
      pick: FinzPick;
      model: string;
      usage: { prompt: number; output: number; thoughts?: number; total: number };
    }
  | { status: "not-configured"; reason: string }
  | { status: "disabled"; reason: string }
  | { status: "error"; reason: string }
  | { status: "invalid-response"; reason: string; raw?: string };
```

### FinzPick 데이터 모델

```ts
type FinzPick = {
  name: string;                 // 예: "디즈니", "AI 반도체"
  kind: "stock" | "theme";
  oneLine: string;              // 한 줄 요약
  whyThisParty: string[];       // 파티 조합과 연결된 추천 이유 2~3개
  debatePoint: string;          // 친구들이 싸울/토론할 지점
  openingQuestions: string[];   // AI 진행자가 바로 던질 첫 질문 2~3개
  conversationSeeds: string[];  // 중간에 대화가 끊겼을 때 던질 소재 3~5개
  rolePrompts: Array<{
    memberName: string;
    role: string;
    prompt: string;             // 이 멤버가 던지면 좋은 한 줄 관점
  }>;
  caveats: string[];            // 투자 조언 아님, 정보 확인 필요 등
};
```

### AI 진행자 모델

V0에서 AI는 채팅방의 한 명처럼 계속 대화를 이끄는 역할을 가진다. 다만 실시간 채팅 인프라는 아직 만들지 않고, 클라이언트의 현재 대화 목록을 stateless API에 보내 다음 진행 멘트를 받는 방식으로 시작한다.

```ts
type FinzConversationMessage = {
  speaker: string;              // "AI" 또는 멤버 이름
  role?: string;                 // 멤버 캐릭터 role
  text: string;
  createdAt: string;
};

type FinzFacilitatorRequest = {
  pick: FinzPick;
  party: FinzPickRequest["party"];
  messages: FinzConversationMessage[];
};

type FinzFacilitatorResponse =
  | {
      status: "ok";
      next: {
        message: string;         // AI 진행자 멘트
        targetMemberName?: string;
        intent: "question" | "challenge" | "summarize" | "reframe" | "next-topic";
      };
      model: string;
      usage: { prompt: number; output: number; thoughts?: number; total: number };
    }
  | { status: "not-configured"; reason: string }
  | { status: "disabled"; reason: string }
  | { status: "error"; reason: string }
  | { status: "invalid-response"; reason: string; raw?: string };
```

AI 진행자는 다음 행동 중 하나를 선택한다.

- **question**: 아직 말하지 않은 사람에게 캐릭터 역할에 맞는 질문을 던진다.
- **challenge**: 방금 나온 의견에 반대 관점이나 리스크를 붙인다.
- **summarize**: 지금까지의 대화를 1~2문장으로 정리한다.
- **reframe**: 논점이 투자 조언/수익률 경쟁으로 흐르면 "대화 소재" 관점으로 되돌린다.
- **next-topic**: 대화가 끊기면 같은 우정주 안의 다른 싸울 포인트를 던진다.

### 프롬프트 원칙

SYSTEM:

- 당신은 친구들의 투자 대화를 여는 FINZ 진행자다.
- 목적은 매수 추천이 아니라 대화 소재 추천이다.
- 실제 투자 조언처럼 단정하지 않는다.
- 수익률, 목표가, 매수/매도 지시를 하지 않는다.
- 파티 멤버의 캐릭터/스탯/태그를 반드시 추천 이유에 반영한다.
- 조용한 멤버가 있으면 그 멤버의 캐릭터 역할에 맞는 질문을 던진다.
- 한쪽 의견으로 쏠리면 반대 관점, 리스크, 확인할 사실을 던진다.
- 대화가 식으면 새로운 뉴스/실적/브랜드/밸류에이션/소비자 관찰 소재를 제안한다.
- 친구들끼리 말할 수 있는 짧고 자연스러운 문장으로 쓴다. 리포트처럼 길게 쓰지 않는다.
- 결과는 JSON만 반환한다.

USER:

- 파티 멤버 목록
- 각 멤버의 role/title/stats/tags/weakness
- 파티 요약 headline/note/tags
- "오늘 이야기하면 재밌을 종목 또는 테마 1개만 추천하고, 바로 던질 질문과 대화가 끊겼을 때 이어갈 소재를 같이 줘"

### Gemini 호출 방식

- 기존 `lib/common/llm.ts` 의 `callLlm` 을 재사용한다.
- 기존 모델 fallback, kill switch, `GEMINI_API_KEY`, `GEMINI_DISABLED`, `PROMPT_VERSION` 정책을 그대로 따른다.
- `responseSchema` 를 사용해 `FinzPick` JSON 형식을 강제한다.
- timeout/fallback/quota 처리는 travel 과 같은 패턴을 따른다.

### 저장소 V0

완전한 친구 초대 MVP까지 가려면 서버 저장이 필요하지만, AI 연결 V0는 두 단계로 나눈다.

1. **AI Pick V0**: 저장 없이 현재 화면의 파티 상태를 API로 보내고 우정주 결과를 즉시 표시한다.
2. **공유 V0**: `FINZ-GROUP-01` 강화 단계에서 Upstash KV에 party snapshot을 저장하고 `/finz/party/[partyId]` 링크로 복원한다.

즉, AI 연결을 먼저 완성하고, 실제 초대 링크/저장소는 그 다음에 붙인다.

### 안전 문구

화면과 응답에는 다음 톤을 유지한다.

- "투자 조언이 아니라 친구들과 이야기할 소재입니다."
- "실제 투자 전에는 가격, 재무, 뉴스, 리스크를 직접 확인하세요."
- "FINZ는 매수/매도 판단을 대신하지 않습니다."

### V0 완료 조건

- 파티 멤버 1명 이상일 때 "오늘의 우정주 뽑기" 버튼이 보인다.
- 버튼 클릭 시 `/api/finz/pick` 이 Gemini를 호출한다.
- 성공 시 `FinzPick` 카드와 AI 진행 질문이 화면에 표시된다.
- 사용자가 한 줄 의견을 남기면 `/api/finz/facilitator` 가 다음 AI 진행 멘트를 반환한다.
- Gemini 키가 없거나 꺼져 있으면 사용자 친화적인 오류를 보여준다.
- `npm run typecheck`, `npm run lint`, `npm run build` 통과.
- 문서상 `FINZ-PICK-01` 상태를 `live` 또는 `partial` 로 갱신한다.

### 5. 투자 레이드

오늘의 우정주를 채팅방에 던지는 대신, 게임 레이드처럼 역할 기반 대화로 만든다.

역할별 대화 미션:

- 딜러: 성장 가능성으로 공격한다.
- 탱커: 리스크와 밸류에이션을 방어한다.
- 힐러: 포트폴리오 안정성을 제안한다.
- 마법사: 금리, 환율, 매크로 흐름을 해석한다.
- 레인저: 실제 소비자와 브랜드 관찰을 제시한다.
- 버서커: 시장의 과열과 밈 에너지를 읽는다.

각 사용자는 자신의 캐릭터에 맞는 한 줄 포지션을 남긴다.

예시 포지션:

- 매수
- 관망
- 회의적
- 모르지만 끌림
- 너무 비싸지만 계속 보게 됨

AI는 대화 후 파티 요약과 다음 힌트를 정리한다.

## MVP 1에서 하지 않을 것

- 실제 매매 기능
- 실시간 주가 차트 중심 화면
- 개인 실제 자산 입력
- 수익률 인증 경쟁
- 복잡한 포트폴리오 관리
- 자동 종목 뉴스 브리핑
- 고급 가격 API 연동
- 공동 모의투자 정산

이유: MVP 1의 목표는 금융 기능 완성도가 아니라 "친구들이 이 주제로 즐겁게 대화하는가"를 검증하는 것이다.

## 이후 확장 순서

MVP 1 사용 후 다음 순서로 기능형 콘텐츠를 검토한다.

1. **투표**: 우정주 후보, 레이드 결론, 다음 레이드 주제를 가볍게 결정.
2. **예측 카드**: 한 주 뒤 결과가 돌아오는 예측 게임. 캐릭터 성장 로그와 연결.
3. **공동 모의투자**: 그룹 가상 포트폴리오. 수익률 경쟁으로 변질되지 않게 "우리 판단 복기" 중심으로 설계.
4. **개인 모의투자 리그**: 경쟁성이 강하므로 MVP 1의 대화 중심 톤을 해치지 않는 보상 구조가 필요.
5. **주간 파티 리포트**: 대화, 포지션, 투표, 예측을 AI가 요약하고 캐릭터 성장에 반영.

## 성공 기준

- 친구 3명이 각자 캐릭터 결과를 보고 웃거나 공유한다.
- 그룹 생성 후 "오늘의 우정주"를 보고 한 줄 이상 의견을 남긴다.
- 레이드 요약을 보고 다음 주제도 해보고 싶다는 반응이 나온다.
- 채팅방이 단순 잡담이 아니라 투자 맥락의 대화로 이어진다.
- 최소 한 명이 "이건 우리끼리 계속 해볼 만하다"고 말한다.
