## 0. 문제의식 — 실사용 중심으로 전환

- 이상적으로 생각했던 "공통 주제로 많이 떠드는" 형태로는 **아직 안 쓰이고 있다**는 공감대. 지금은 우리끼리 테스트 과정이라 괜찮지만, 어떻게 하면 공통 주제로 더 많이 대화하게 할지 고민 필요.
- **B**: 공통 주제로 떠들려면 결국 **우정주가 확정**되고 그걸 중심으로 이야기해야 하는데, 아직 우정주 확정 흐름이 없어 대화가 한곳으로 안 모임. 그걸 구현하면 대화가 모일 것.
- **A**: 기능만 만들어 쑤셔넣다 보니 **기능 테스트만 하는 느낌**. 여기까지를 기능 MVP 일단락으로 보고, 이제 **서사(대화)를 만들어가는 데 집중**하자. 대화가 쌓이면 그걸로 회고·분석이 가능해짐.

> "기능 추가에 집중하는 게 아니라 대화에 집중해보자."

## 1. 우정주 정의·도출·활용 시나리오

- **우정주란**: (예전 조메디카처럼) 우리만의 공통 종목. 비중이 크지 않아도, 함께 움직임·소식을 보며 대화하는 계기가 되는 "공통점".
- **도출 방식**: 우리가 직접 고를지 vs 핀즈가 추천할지 논의 → **핀즈가 후보를 정리·추천 → 우리가 대화로 확정**하는 방향.
- 우정주까지 가는 **"서사"**(누적된 메시지·토론)가 핵심. 누가 종목/테마를 던지고 → 서로 의견을 주고받고 → "같이 투자해보자"로 결론이 지어지는 과정.

## 2. 슬랙식 스레드/리플라이 구조

- **A**가 슬랙 화면을 공유하며 설명: 왼쪽 리스트가 채널(=대화방), **메시지 하나하나가 "스레드"**, 그 스레드 아래 **"리플라이"**로 그 주제에 대해서만 대화가 묶임. 플랫하게 풀어 쓰다가 특정 주제에 리플라이로 파고들 수 있음.
- **장점**: 채팅처럼 주제가 밀려 휘발되지 않고 **각 주제가 살아있음**. 여러 주제를 동시에 진행해도 각각 유지. 이미 주제 안에 묶여 있어 **AI가 주제별로 묶어내기 유리**.
- **전제/단점**: 팀이 "어디에 리플라이할지"에 대한 **공유된 습관**이 있어야 함. 안 써본 사람은 새 메시지를 계속 만들어 주제가 흩어짐(슬랙엔 스레드별 고유 링크로 연결하는 문화가 있음).
- **아이디어**: 집중 토론방을 따로 두기보다, **애초에 대화방 자체를 스레드 형태**로 구성.
- **(회의 후반) 스레드 모드 배포 완료 공유** — 방마다 **일반 채팅 / 스레드** 모드를 전환할 수 있게 배포됨. 데이터 구조는 그대로 두고(이질감 최소화) **표현 방식만** 바뀜: 일반 모드에선 카카오톡식 말풍선, 스레드 모드에선 원글 + 답글. (서치 기능도 필요하다는 메모.)

## 3. AI 자동 카테고리 분류 (아이디어)

- **B**: 슬랙처럼 사람이 수동으로 분류하지 말고, **우리 대화를 AI가 분석해 주식 카테고리별로 자동으로 묶기**. "너네가 제일 관심 많은 게 이거야" 식으로 도출 → 그 중 탑 랭킹을 "베스트 우정주"로 끌고 갈 수도.
- **A**: **리포트 형태**에서 유용해 보이는 기능. 다만 실시간 분류는 **토큰 비용**이 커서, 리포팅 시점에 처리하는 편이 적합.

## 4. 컨텍스트 엔지니어링 · 스냅샷 저장

- **A**: @finz(AI) 기능을 급하게 붙이느라 엔지니어링을 거의 안 하고 AI에 맡겨서 **대화가 어색**함. LLM은 **매 요청마다 이전 맥락을 다 실어 보내야** 함(요청 간 연결성 없음 = 시스템 프롬프트 + 기존 컨텍스트 + 새 메시지를 매번 조립).
- **B**: 요약 기능을 넣으려면 대화를 저장해야 한다고 (코덱스가) 계속 물어봄. 저장 시점부터 **누적 관리**하면 매번 전체를 안 보내도 됨.
- **A**: 메시지 자체는 이미 DB에 저장돼 있음. 코덱스가 말하는 건 **스냅샷**(요약본 저장) — 요약할 때 전체 메시지 대신 **이전 요약 + 새 메시지만** 넣어 다시 요약 → 요약이 누적되는 구조.
- **결정 여지**: 리포트를 요약본 / 풀본 / 일부만 요약 중 선택할 수 있게.

## 6. 실사용 강화 — 모의투자 / 대회 / 단기 목표

- **A**: **모의투자** 개념을 도입해 진입장벽을 낮추자. 이미 **포트폴리오 기능**이 있어 매수/매도를 기록하면 성과가 자동 산출됨 → 그 성과로 대화가 이어짐(수익률 자랑, 손실 회고 등).
- **C**: 단순 기록은 귀찮아서 안 이어짐. **기능 강화** 필요 — 투자방을 만들고 투자금을 정해, 참여자 수익·거래 횟수가 서로 **공유**되는 형태여야 진지하게 참여.
- **A**: 암호화폐 거래소식 **모의투자 대회**(기간 정하고, 수익금 또는 수익률 기준 등수) → 기존 **피드 기능**으로 결과 공유("이번 대회 ○○ 1등, 어떤 종목으로 몇 회 거래, 총 수익 얼마"). 처음엔 모의투자만, 나중엔 실투자 검증까지. **영웅심리** 활용.
- **C**: 자극적으로 가려면 단기 반등 종목을 AI가 계속 **리스트업** → 우리가 분석·의견. 다만 "그렇게까지 해야 하나 / 건강한 방식인가" 우려도.
- **B·C**: 대회로 가면 **우정주에서 멀어져 단순 트레이딩**이 될 우려 → 방향(우정주 관리 vs 전략)을 정해야 함.
- **C**: 우정주에 **단기 목표**(예: 2주 내 +20%, 손절 -5%)를 함께 설정. 목표가 있어야 장기 보유도 재미·긴장감이 생김. 달성하면 다음 우정주로 교체.

## 7. 전략 vs 포트폴리오 기반 기능

- **A**: 전략(언제·얼마에 매수/매도, 분할매수·분할매도)은 무궁무진하지만, 이를 **지원하는 "기능"은 매수·매도 기록이 핵심**. 전략을 저장하려면 그 정보를 받아줄 기록 기능이 필요. (몇 % 오르면 매수, 얼마 되면 매도 등도 결국 전략.) → 사실상 자동매매의 기초.
- **A**: 이미 만든 **공동 포트폴리오** 기능에 **평단 자동 계산**이 있음. 메시지로 `"테슬라 1주 매수, 포트폴리오에 추가해줘"` → 공동 포트폴리오 반영, 평가손익 표시. 설정에서 수동 거래 추가도 가능.
- 정리: **기반 기능은 이미 구현**돼 있어, 여기에 확장하거나 새 판을 깔면 됨. 자동매매(트리거 매수/매도)·분할 전략까지 확장 가능하나 v1 과범위는 주의.

> "이걸 계속 얘기하려던 게, **기반이 되는 기능은 이미 만들어 놨다**는 것."

## 8. 우정주 추천→확정→단기 목표→데일리 리포트 (기능 정리)

흐름을 컴팩트하게 정리:

1. **추천**: 플러스 버튼 → "우정주 추천" → 항목(**기간 / 목표 수익률 / 카테고리** [+ 뽑은 카드 반영 여부]) 입력 → 핀즈가 **후보 N개** 제시.
2. **확정**: 후보 중 **대화로 합의**해 선택 → 등록. (너무 타이트하지 않게, 대화방 스타일로 자연스럽게 확정.)
3. **단기 목표**: **매수가 / 익절가(목표 수익률) / 손절가 / 투자 기간** 설정. (분할매수·분할매도는 후속 제안 가능.)
4. **데일리 리포트**: 등록 종목의 **가격·뉴스·차트**를 매일 특정 시간 자동 발송 → 대시보드로 성과 확인. 매일 아침 대화 거리가 생김.

## 9. 우정주 추천 UI — 입력 방식

- **C**: 텍스트로 치는 대신 **플러스 버튼 메뉴에서 항목화**하고 싶다.
- **A**: 폼을 완전히 고정하면 **재사용성↓·오히려 불편**(날짜 상자 여러 개 누르기 등). LLM이 자연어 맥락 분석을 잘하니 **자율성**을 주는 게 나음. 규격이 있어야 확실히 동작하는 정보를 전달.
- **절충안(합의)**: **문항(항목)은 구분해 띄우되, 입력은 주관식(자연어)** — 코덱스처럼 "항목 제시 + 텍스트 입력". 후보 선택은 **버튼/체크**로 눌러 바로 포트폴리오에 추가.

## 10. 합의(우정주 확정) 프로세스

- 후보/옵션은 **누구나 만들 수 있고 다 같이 수정** 가능하되, **변경 시 합의(승인) 초기화**(커플렛 방식)를 논의.
- 다만 가벼운 채팅방엔 강한 합의가 과할 수 있음 → 결국 **시스템적 합의보다 대화로 자연스럽게 확정**하는 쪽으로 기울음(등록은 느슨하게, "등록해줘" 하면 바로 등록).

## 11. 기타 — 월간 리뷰 오류 / 기여 방식

- 배포 중 확인: **월간 리뷰 기능 테스트 실패(깨짐)** → 수정 필요. **B**가 오류 링크/캡처를 코덱스에 전달해 수정 요청하기로.
- 기여: 틈틈이 개별 피처 수정·개선 의견은 **카카오톡으로 공유**하고, 동의 시 반영.

## 주요 결정사항

- **실사용 중심 전환**: 기능 MVP는 일단락, 이제 대화(서사)를 쌓는 데 집중.
- **우정주 추천 방식**: 핀즈가 기간·목표 수익률·카테고리 조건을 받아 후보 리스트 제공 → **대화로 합의해 확정**.
- **단기 목표 설정**: 우정주 확정 시 매수가·손절가·목표 수익률(예: 10~20%)·투자 기간 함께 설정.
- **데일리 리포트 자동화**: 등록 종목 가격·뉴스·차트를 정기적으로 채팅방에 자동 발송.
- **스레드 모드 배포 완료**: 슬랙식 스레드 UI 배포 — 방별 일반/스레드 전환(데이터 공유).
- **추천 UI**: 항목은 구분해 띄우되 입력은 주관식(폼 고정 지양), 후보는 버튼/체크 선택.
- **우정주 확정은 느슨하게**: 시스템 강제 합의가 아니라 대화 기반으로 자연스럽게.
- **토큰 최적화는 후순위**: 개발 단계에선 제한 없이 기능 우선.
